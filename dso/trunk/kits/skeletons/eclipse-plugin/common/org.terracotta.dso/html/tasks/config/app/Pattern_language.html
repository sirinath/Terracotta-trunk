Class Expression Patterns

The pattern language for specifying the set of classes to be instrumented or excluded is based on those used in Aspect-oriented Programming frameworks for specify pointcuts. The following are examples of valid class instrumentation include and exclude patterns:

com.project.Main class Main in package com.project
com.project.* all classes in package com.project
com.project.*Main all classes in package com.project ending with Main
com..* all classes under com

Method Expression Patterns

The pattern language for specifying the set of methods to be locked or distributed is based on those used in Aspect-oriented Programming frameworks for specify pointcuts. The following are examples of valid method specifier patterns:

void com.project.Main.foo() method foo() in class com.project.Main with no arguments or return value
* com.project.Main.foo() all methods foo() in class com.project.Main with no arguments
boolean com.project.Main.foo(..) all methods foo() in class com.project.Main returning boolean
java.lang.* com.project.Main.foo*(int,..) all methods in package com.project.Main beginning with foo, whose first argument is an int, returning a type defined in package java.lang

For the complete description refer to the Aspectwerks joinpoint selection pattern language.
