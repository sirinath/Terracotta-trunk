Terracotta Developer Console: Topology

Contains detailed information about the clients and server arrays. Mirror Groups (sets of active-"hot" standby server instances) are listed under Server Arrays, while Terracotta server instances are listed under their parent mirror group. Each server instance has a colored status icon:

Greenactive server
Redunreachable
Yellowstarting or disk-based standby
Orangeinitializing (network-based only)
Cyannetwork-based standby
Connected Clients
View a table of clients currently in the cluster. The table, which automatically refreshes, displays a live-object count per client. Choose a client in the navigation list to view information panels about that client.
Server Array
Cluster server arrays are arranged in Terracotta mirror groups (sets of active-"hot" standby server instances), each containing individual Terracotta server instances. Choose a server-array to display a table of its mirror groups. Select a mirror group to display a table of its server instances. Select a server instance to display information panels about that server instance.
A warning appears in the Main server-instance panel if it is not configured to persist data across failures. This mode is useful in certain development environments to meet data storage constraints.
