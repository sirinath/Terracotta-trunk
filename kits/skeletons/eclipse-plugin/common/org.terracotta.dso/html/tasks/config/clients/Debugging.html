Client Debugging Options

A Terracotta client can emit various debugging message to its log. There are three categories of such debugging messages: (1) Instrumentation logging, (2) Runtime logging, and (3) Runtime output options.

Instrumentation Logging

The following options produce output in response to particular configuration settings.
Class Class is being instrumented
Locks Locks declared on an instrumented class
Transient root Warns that a declared root is also declared as a transient field
Roots Roots declared on an instrumented class
Change listener Change listeners declared on an instrumented class
Distributed methods Distributed methods declared on an instrumented class

Runtime Logging

The following options produce output while DSO is managing object state.
Lock debug Traces lock aquisitions
Field-change debug Traces changes to shared objects
Non-portable warning Attempt to add a non-portable object to shared object graph
Wait-notify debug Traces synchronization contention points
Distributed method debug Traces distributed method invocations
New object debug Indicates the creation of a new shared object

Runtime Output Options

The following options produce extra runtime tracing output, extending other Runtime Logging options.
Auto-lock details If Lock debug enabled, displays the class name and identityHashCode of the instance being synchronized on
Caller Extends Lock, Distributed method, and New object debug with the triggering (top) stack frame
Full stack Extends Caller to output the full stack
